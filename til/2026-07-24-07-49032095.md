# TIL: PImpl Idiom and C++26 std::indirect_type

Today I learned about the implications of the PImpl idiom's evolution in C++26, thanks to the introduction of `std::indirect_type`. This new type allows for more flexible and expressive PImpl implementations by decoupling the pointer-to-member and pointer-to-data relationships. The `std::indirect_type` class template enables the use of `std::variant` and other advanced types in PImpl, potentially leading to improved code organization and reduced dependencies. While it may not revolutionize the way I write PImpl today, understanding the new possibilities and implications of `std::indirect_type` will help me better navigate the evolving landscape of C++ and design more maintainable systems.
